    dispatcherId

    You may use akka.testkit.CallingThreadDispatcher.Id here for now when testing an actor that uses pipeTo, to make sure messages are delivered before collecting side effects and continuing the test. Better might be to use an asynchronous dispatcher which can be inspected to make sure all pending tasks have completed before continuing, but no such dispatcher is provided at this time.
