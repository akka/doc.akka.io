Companion object Props

abstract class Props extends Product with Serializable

Data structure for describing an actor’s props details like which executor to run it on. For each type of setting (e.g. DispatcherSelector or MailboxCapacity) the FIRST occurrence is used when creating the actor; this means that adding configuration using the "with" methods overrides what was configured previously.

Deliberately not sealed in order to emphasize future extensibility by the framework—this is not intended to be extended by user code.
